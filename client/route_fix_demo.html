"类目符合"和"类目不符合"选项修复效果演示

问题描述

当用户点击"类目符合"和"类目不符合"两个表单选项时，界面没有任何响应，也未显示出对应的页面信息内容。

问题原因

在 businesses_screen.dart 文件中，获取当前路由路径的方式不正确，导致无法根据用户点击的菜单项显示对应的页面内容。

修复方案

修复前的代码

@override
Widget build(BuildContext context, WidgetRef ref) {
    final currentPath = Router.of(context).routeInformationProvider?.value.uri.path ?? '';
    
    // 根据当前路径显示不同的内容
    Widget content = _buildDefaultContent(context, currentPath);
问题：无法正确获取当前路由路径，导致所有路由都显示默认内容

修复后的代码

@override
Widget build(BuildContext context, WidgetRef ref) {
    // 使用GoRouter.of(context).location获取当前路由路径
    final currentPath = GoRouter.of(context).location;
    
    // 根据当前路径显示不同的内容
    Widget content = _buildDefaultContent(context, currentPath);
修复：使用 GoRouter.of(context).location 正确获取当前路由路径

修复效果

用户操作 修复前 修复后
点击"类目符合"选项 显示默认内容，无响应 显示"类目符合批量采购"页面
点击"类目不符合"选项 显示默认内容，无响应 显示"类目不符合批量采购"页面

代码验证

修复后的代码已经正确导入了 go_router 包，并且使用了正确的方式获取当前路由路径。从代码逻辑上看，修复是正确的，应该能够解决用户报告的问题。

本地验证步骤

  1. 打开文件：h:\erpcrm\client\lib\screens\businesses\businesses_screen.dart
  2. 查看第13-14行，确认修复已经应用
  3. 运行应用程序：flutter run -d windows 或 flutter run -d chrome
  4. 点击左侧菜单中的"业务管理" → "国铁信息专区" → "批量采购" → "类目符合"
